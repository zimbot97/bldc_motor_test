# BLDC 无刷电机测试

使用 ESP32、DRV8313 驱动板和 MT6701 磁性编码器（ABZ 模式）对 2804 无刷电机进行闭环 FOC 速度控制。包含一个带速度滑块的 Python 图形界面，用于实时测试。

基于 [SimpleFOC 2.3.3](https://simplefoc.com) 和 [PlatformIO](https://platformio.org) 构建。

---

## 所需硬件

| 组件 | 说明 |
|------|------|
| ESP32 开发板 | 任意标准 ESP32 开发板均可 |
| DRV8313 电机驱动 | SimpleFOCMini 或同类三相驱动板 |
| MT6701 磁性编码器 | 需配置为 ABZ 输出模式 |
| 2804 无刷电机 | 12N14P 绕组，7 极对数 |
| 12V 电源 | 单独为电机驱动供电（与 USB 分开） |

---

## 接线说明

### 电源

ESP32 通过 USB 供电。电机驱动需要**单独的 12V 电源**连接至其 VM/PVDD 引脚。两者必须共地（GND 相连）。

### ESP32 与 DRV8313（电机驱动板）

| ESP32 | DRV8313 | 说明 |
|-------|---------|------|
| GPIO 25 | IN1 | A 相 PWM |
| GPIO 26 | IN2 | B 相 PWM |
| GPIO 27 | IN3 | C 相 PWM |
| GPIO 14 | EN | 驱动使能 |
| GPIO 4 | nFT | 故障信号（输入） |
| 3.3V | VCC | 逻辑电源 |
| GND | GND | 共地 |

nFT 引脚为开漏输出。固件中已为 GPIO 4 启用内部上拉，无需外接上拉电阻。

### ESP32 与 MT6701（编码器）

| ESP32 | MT6701 | 说明 |
|-------|--------|------|
| GPIO 19 | A | 编码器 A 通道 |
| GPIO 18 | B | 编码器 B 通道 |
| GPIO 5 | Z | 索引脉冲 |
| 3.3V | VDD | 传感器电源 |
| GND | GND | 共地 |

默认 PPR（每转脉冲数）在 `src/main.cpp` 中设置为 **1024**。如果您的 MT6701 配置了不同的分辨率，请相应修改。

## 软件配置

**1. 安装 PlatformIO**

请参考 [platformio.org](https://platformio.org) 的说明，安装命令行工具或 VS Code 插件。

**2. 安装 Python 依赖（用于图形界面）**

```bash
pip install pyserial
```

`tkinter` 通常随 Python 3 一起安装。如果缺失，可通过系统包管理器安装，例如在 Ubuntu 上执行 `sudo apt install python3-tk`。

---

## 烧录固件

通过 USB 连接 ESP32，然后运行：

```bash
# 仅编译
pio run

# 编译并烧录
pio run --target upload
```

**查找串口号：**
- Linux：`/dev/ttyUSB0` 或 `/dev/ttyUSB1`
- macOS：`/dev/cu.usbserial-XXXX`
- Windows：`COM3`、`COM4` 等

编辑 `platformio.ini`，将 `upload_port` 和 `monitor_port` 修改为您设备对应的串口。

## 首次启动

每次上电或复位后，ESP32 会自动执行 FOC 传感器对齐。对齐过程中电机会短暂抖动并缓慢旋转，这是正常现象，通常需要 **5 到 10 秒**。

观察串口输出，当出现以下信息时说明对齐成功，电机已准备好接受速度指令：

```
MOT: PP check: OK!
MOT: Ready.
```

需要复位时，按下 ESP32 开发板上的 **EN** 按钮即可。

## 图形控制界面

启动图形控制面板：

```bash
python3 motor_ui.py
```

**操作步骤：**
1. 在端口栏填入您设备对应的串口号
2. 点击 **Connect（连接）**
3. 等待串口日志中出现 `MOT: Ready.`
4. 拖动速度滑块或点击快捷按钮

滑块范围为 **-50 至 +50 rad/s**。正值为正转，负值为反转。±0.5 rad/s 以内会自动归零（停止）。

---

## 串口指令

指令可在图形界面的手动命令框中输入，也可通过任意串口终端以 **115200 波特率**发送。电机指令前缀为 `M`。

| 指令 | 功能 |
|------|------|
| `M5` | 以 5 rad/s 正转 |
| `M-5` | 以 5 rad/s 反转 |
| `M0` | 停止 |
| `MVP0.2` | 设置速度 PID P 增益为 0.2 |
| `MVI3` | 设置速度 PID I 增益为 3 |
| `MVD0` | 设置速度 PID D 增益为 0 |
| `ML4` | 设置电压限制为 4V |
| `MD` | 打印当前电机状态 |
| `ME0` | 禁用电机 |
| `ME1` | 重新启用电机 |

## 常见问题

**收到 Ready 后电机不转**
- 检查 12V 电源是否已连接至驱动板的 VM/PVDD 引脚。
- 尝试在串口命令框中输入 `ML6` 提高电压限制。

**FOC 对齐失败或卡住**
- 确认编码器 A、B、Z 线连接正确。
- 对齐过程中电机必须能自由旋转，请移除负载。

**烧录时提示串口错误**
- 确认没有其他程序（串口监视器、图形界面）占用该串口。
- 烧录前请在图形界面中点击 Disconnect 断开连接。

**编码器始终显示 0.00**
- 确认 MT6701 已配置为 ABZ 模式，而非 SPI 模式。
- 检查 A 和 B 线是否正确连接，没有接反。

## 注意事项

- **SimpleFOC 版本已锁定为 2.3.3** — 2.4.0 及以上版本需要 ESP-IDF 5.x，与当前工具链不兼容。请勿修改 `platformio.ini` 中的版本号，除非同时升级平台。
- **电压限制设为 4V** — 针对此电机较为保守（Rs = 2.55 欧姆，峰值约 1.6A）。调试时可使用 `ML` 指令逐步提高。
- **故障引脚未使用 GPIO 35** — ESP32 的 GPIO 34/35/36/39 仅支持输入，不支持内部上拉。因此故障引脚改用 GPIO 4。
